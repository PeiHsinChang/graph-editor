# graph-editor

- 安裝 pnpm
- 主要工具 Vite + React + TypeScript + Tailwind

🟢 Day 1：基礎建設與狀態驅動 (已完成！🎉)
**🎯 目標：建立乾淨的專案環境，並能成功在畫布上透過狀態 (State) 渲染節點。**

**工程環境架設：** 成功使用 pnpm + Vite + React + TypeScript + Tailwind 打造出極輕量且高效的開發環境。

**資料結構定義：** 設計出 Clean Code 友善的正規化資料結構，明確區分 Node 與 Edge 的型別。

**畫布座標轉換：** 實作 getBoundingClientRect() 來精準校正滑鼠與畫布的相對座標。

**事件隔離機制：** 成功處理 DOM 事件冒泡 (e.stopPropagation())，做到「點空白處新增節點」與「點節點選取」互不干擾。

**UI 視覺升級：** 加入了網點背景 (Dot Grid) 與屬性面板，打造出專業工具的質感。

🟡 Day 2：核心互動 —— 絲滑拖曳 (目前進度 🚀)
**🎯 目標：讓畫布「活」起來，打破靜態畫面，實作高互動性的拖曳體驗。**

**全域事件監聽：** 將 onMouseMove 與 onMouseUp 綁定在最外層 <svg>，防止滑鼠移動太快導致拖曳斷開。

**效能優化追蹤：** 運用 useRef 來記錄當前拖曳的節點 ID，避免觸發多餘的畫面重繪。

**狀態即時更新：** 滑鼠移動時，動態更新目標節點的 (x, y) 座標，讓 SVG 畫面即時重繪。

**刪除機制 (準備進行)：** 加上鍵盤監聽（例如 Delete 或 Backspace 鍵），實作刪除被選取節點的邏輯。

🟠 Day 3：拓撲靈魂 —— 動態連線 (Edge Creation)
**🎯 目標：實作點與點之間的關係建立，完成圖論編輯器的最後一塊拼圖。**

**狀態機 (State Machine) 切換：** 在 UI 面板新增按鈕，讓游標可以在「選取 / 拖曳模式 (SELECT)」與「連線模式 (ADD_EDGE)」之間切換。

**視覺預覽線 (Preview Line)：** 在連線模式下，點擊起點節點 (Source) 後，畫布上會生出一條「跟著滑鼠游標移動」的虛線，提供強烈的互動回饋感。

**關係綁定：** 點擊終點節點 (Target) 後，正式生成一筆 Edge 資料，並消滅預覽線。

**終極連動測試：** 驗證拖曳節點時，綁定在它身上的線條必須完美無瑕地跟著伸縮變形。
